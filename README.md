협업 시 주의사항 (도커 기반)

1. 패키지 업데이트 시 재빌드

* package.json이 변경된 경우 git pull 후에 반드시 docker-compose up --build를 실행해야 합니다.
* 단순 pull만 하면 컨테이너 내부 라이브러리가 업데이트되지 않아 에러가 발생합니다.

2. 환경 변수 동기화

* .env 파일은 보안상 깃허브에 올리지 않으므로, 환경 변수를 새로 추가하면 팀원에게 직접 내용을 공유해야 합니다.
* 내용을 전달받은 팀원은 로컬의 .env를 수정한 후 도커를 재시작해야 적용됩니다.

3. 로컬 node_modules 주의

* 로컬 PC에 직접 npm install을 해서 생긴 node_modules가 도커 컨테이너 내부 설정과 충돌할 수 있습니다.
* 가급적 로컬의 node_modules는 삭제하거나 비워두고, 모든 실행은 도커를 통해서만 진행합니다.

4. 도커 설정 변경 공유

* docker-compose.yml이나 Dockerfile을 수정하여 푸시했을 경우 반드시 팀원에게 공지합니다.
* 설정을 바꾼 후에는 컨테이너를 내렸다가(down) 다시 올리는 과정이 필요할 수 있습니다.

5. 포트 충돌 방지

* 로컬에서 개별적으로 node를 실행하여 3000번 포트를 점유하고 있으면 도커 컨테이너가 실행되지 않습니다.
* 작업 전 3000번 포트가 비어있는지 확인하십시오.
